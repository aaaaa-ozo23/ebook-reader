# Stage 13.3 Data & Backup fidelity ledger

The settings surface extends the approved Stage 13 visual contract rather than introducing a
second product style. It uses the existing warm-paper workspace, deep-ink structure, teal state,
amber focus, thin borders, and restrained material depth.

| Before | After | Why |
| --- | --- | --- |
| No product settings entry | Shelf rail adds a direct **Settings** destination; 375 px adds a 44 px header control | Keeps wayfinding visible at both desktop and touch breakpoints |
| Data actions had no dedicated surface | Desktop uses a deep-ink settings sidebar and open warm-paper workspace | Preserves the approved structural hierarchy without dashboard-style cards |
| No mobile data-management pattern | Settings becomes a fixed, scrollable full-screen sheet with a persistent **Back to shelf** action | Prioritizes readable copy and safe touch targets over compressing desktop navigation |
| Backup content was only a technical contract | Options use calm rows with clear default states, size-risk copy, and an unencrypted-data notice | Makes privacy and storage consequences visible before the file dialog |
| Long-running state had no shared visual language | Export uses a structured progress track, explicit cancel, success/error result, and no hidden completion | Preserves user agency and maps directly to the operation registry |
| Generic motion could delay repeated work | Press feedback is 0.97/120 ms, sheet entrance is 220 ms, keyboard paths are immediate, reduced motion removes translation | Matches the approved Emil/Apple motion constraints |

## Verified comparison points

1. Desktop settings sidebar reuses `#1F3035` structural chrome and the approved ER amber mark.
2. Selected navigation and progress use `#235F62`; focus uses the existing 3 px amber ring.
3. 1280 × 720 keeps the primary export action in the first viewport with no page scroll.
4. 900, 640, and 375 widths have no horizontal overflow; 375 renders as a full-screen sheet.
5. All actionable touch targets are at least 44 px and axe reports no serious/critical issues.
6. The core/covers/books defaults, unencrypted notice, cache exclusion, and desktop-runtime error
   are visible product copy rather than documentation-only behavior.

No new raster artwork was required; the approved design system's existing vector icon language is
sufficient for the settings and backup states.
