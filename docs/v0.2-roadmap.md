# Ebook Reader v0.2 路线图

状态：大阶段 8 规划基线

更新日期：2026-07-02

适用版本：v0.1.0 之后的 v0.2 开发周期

## 1. 目标与边界

v0.2 的目标是把 v0.1.0 已验证的本地书库、三格式阅读、进度、书签、标注和搜索能力，扩展为更完整、可持续演进的阅读体验。首要原则是保留稳定 locator 和现有数据兼容性，不让临时页码、动画实现或具体渲染库进入持久化模型。

大阶段 8 只完成研究、架构决策和实施排序，不实现功能。阶段 8 不允许：

- 修改应用代码、依赖、数据库 schema、应用版本或 `CHANGELOG.md`。
- 创建 `codex/v0.2.0-integration`；该分支在阶段 9 实施开始时从最新 `main` 创建。
- 把尚未实现的能力写成 README 中的现有功能或发布承诺。
- 同时启动 MOBI/AZW3、跨平台、移动端、云同步等会扩大 v0.2 关键路径的工作。

## 2. 当前架构基线

现有架构可以承载 v0.2，但需要先补齐统一的阅读体验偏好和能力描述：

- `TxtLocator.charOffset`、`EpubLocator.cfi/href`、`PdfLocator.page` 已是稳定定位锚点。
- `app_settings` 是键值 JSON 存储，可增加全局阅读体验设置而无需 schema migration。
- TXT 已使用 TanStack Virtual；PDF.js 已通过自定义 Canvas、文本层和标注层渲染；EPUB 使用 epub.js 分页 rendition。
- `PdfViewMode` 已声明 `continuous`，但当前实际渲染只落到 single/double。
- EPUB 当前底部页码来自每 1500 字符生成的 locations，不是出版物 `page-list`。
- `ReaderShell.tsx` 同时承担布局、设置、三种阅读器、选择菜单和标注交互；后续应分模块渐进拆分，不能一次性重写。

## 3. 共享接口与兼容策略

### 3.1 计划新增的共享类型

阶段 9 实施以下公共类型；名称在实现阶段保持不变：

```ts
export type PageTransitionMode = "none" | "slide" | "page-curl";
export type EpubViewMode = "paginated";
export type TxtViewMode = "scroll" | "paginated";
export type PdfViewMode = "single" | "double" | "continuous";

export interface ReaderCapabilities {
  viewModes: readonly ReaderViewMode[];
  pageTransitions: readonly PageTransitionMode[];
  supportsPublicationPageLabels: boolean;
  supportsImageViewer: boolean;
}

export type ReaderViewMode = EpubViewMode | TxtViewMode | PdfViewMode;

export interface ReaderExperiencePreferences {
  epub: {
    viewMode: EpubViewMode;
    transition: PageTransitionMode;
  };
  txt: {
    viewMode: TxtViewMode;
    transition: PageTransitionMode;
  };
  pdf: {
    viewMode: PdfViewMode;
    transition: PageTransitionMode;
  };
}
```

默认值固定为：

```ts
{
  epub: { viewMode: "paginated", transition: "slide" },
  txt: { viewMode: "scroll", transition: "slide" },
  pdf: { viewMode: "single", transition: "slide" }
}
```

`prefers-reduced-motion: reduce` 只把运行时有效过渡降级为 `none`，不覆盖用户保存的偏好。PDF `continuous` 模式的有效过渡始终是 `none`。

### 3.2 Locator 兼容

- `TxtLocator` 和 `EpubLocator` 不变；分页结果和出版物页码都不能成为唯一 locator。
- `PdfLocator` 增加可选 `pageOffsetRatio?: number`，范围为 `0..1`，表示连续模式下当前页内的垂直锚点。
- 旧 PDF locator 没有 `pageOffsetRatio` 时按 `0` 恢复到页首。
- 新设置保存到 `app_settings` 的版本化 JSON 值，不新增表或列。
- 阅读模式切换必须先取得当前稳定 locator，再改变布局并恢复该 locator，不能按旧页号猜测位置。

### 3.3 动画控制层

动画属于阅读 UI，不进入 `ReaderAdapter` 的持久化模型。统一控制器遵守以下状态机：

1. 接收前进或后退请求并锁定一次导航事务。
2. 取得当前展示层和目标展示层，真实 reader 在后台完成一次 locator 跳转。
3. `slide` 使用 CSS/WAAPI，目标时长 180–240ms。
4. `page-curl` 使用隔离展示层，目标时长 420–560ms；不得直接包裹可选择、可标注的实时 DOM/iframe。
5. 动画完成后只提交一次进度并移除展示层。
6. 动画中最多保留一个待处理方向，后来的输入覆盖旧待处理输入，避免无限队列。
7. reduced motion、渲染失败或展示层不可捕获时立即无动画完成导航。

真实翻页实现先在阶段 9 做隔离原型，比较自研 CSS 3D 展示层与 `page-flip`。只有同时满足以下条件才引入依赖：

- MIT 兼容、无额外运行时依赖，生产包增量可接受。
- 支持 HTML 与 Canvas 展示层，不要求接管 epub.js iframe 或实时标注 DOM。
- 30 次连续翻页无丢页、重复提交、选区残留或焦点丢失。
- 1280×800 和 640×640 下动画过程中无超过 50ms 的可归因长任务。
- reduced motion、键盘操作和自动化测试能够稳定控制。

任一条件失败则使用自研展示层，不把库强行接入阅读器。

## 4. EPUB 增强

### 4.1 出版物自带页码

EPUB `page-list` 是可选导航结构，表示印刷版或静态分页来源的页面边界。实现策略：

- 读取导航文档中的 `page-list`，保留原始标签，不把罗马数字、前言页码或其他合法标签强制转成整数。
- 将 page-list 目标解析为 href/fragment/CFI 边界，并建立按 spine 顺序排序的查找表。
- 当前 CFI 位于两个边界之间时显示前一个有效出版物页码。
- 有 page-list 时状态文案为 `Page <label>`；无 page-list 时显示 `Location <current> / <total>`。
- 现有基于 locations 的总量继续服务进度滑杆和位置回退，但 UI 不再把它称为出版物页码。
- 数字、罗马数字、href fragment、CFI、空 page-list 和损坏 page-list 都要有 fixture。

风险：epub.js 0.3.93 的 PageList 实现会对标签执行整数解析，并且 href-only 边界不能完全依赖 `pageFromCfi`。因此实现应在 adapter 层保留原始导航标签和边界，不直接把 epub.js 的整数结果作为最终 UI 数据。

### 4.2 EPUB 图片查看器

在 epub.js 每个已渲染 iframe 中注册事件代理，不修改原始 EPUB 文件：

- 鼠标单击以及图片获得焦点后的 Enter/Space 打开查看器。
- 支持 HTML `img` 和 SVG `image`；忽略装饰性空资源和无法解析的目标。
- 使用当前内容已经解析的 `currentSrc`/资源 URL，不为查看器主动访问新的外部 URL。
- 模态框包含关闭、缩小、放大、适应窗口、100%、重置按钮；缩放范围 100%–500%。
- 支持滚轮/按钮/触控缩放、放大后拖动、双击切换 100% 与适应窗口、Esc 关闭。
- 打开时移动焦点，关闭后恢复到原图片；背景不可交互，状态变化通过可访问名称表达。
- 离开书籍时移除 iframe listener 和临时资源引用。

v0.2 不实现图片编辑、导出、OCR、章节图库或主动下载远程图片。

### 4.3 EPUB 翻页动画

- EPUB 保持 `paginated`，不在 v0.2 增加 EPUB 连续滚动。
- single/double spread 都支持 `none`、`slide`、`page-curl`。
- 双页模式以整个 spread 为一次导航事务，不能分别提交左右页。
- 切换主题、窗口尺寸或 spread 时取消动画并用 CFI 恢复。
- 选择菜单、批注编辑器或图片查看器打开时，不响应阅读区翻页快捷键。

## 5. TXT 分页模式

### 5.1 模式和定位

- 连续滚动保持默认，新增 `paginated`。
- 页边界是由当前 viewport 和主题计算出的临时 `charOffset` 区间。
- 主题、字体、行高、边距、窗口尺寸或 DPI 改变时重新分页，并恢复到当前页第一个可见字符附近。
- UI 可以显示当前布局下的 `Page x / y`，但数据库仍保存 `chapterId + charOffset`。

### 5.2 分页器

- 对段落和标题做离屏测量；长段落用 Range/二分法寻找可容纳字符边界。
- 只挂载前页、当前页和后页，保留较小预取窗口，不渲染整本 TXT。
- 缓存键包含书籍 file hash、viewport 尺寸、DPI 和主题排版指纹；任一值改变即失效。
- 标注继续使用全局字符范围，分页切片只负责把范围映射到当前页。
- 章节跳转、搜索、书签和批注跳转统一先定位 charOffset，再由分页器选择页面。

### 5.3 交互

- 滚动模式沿用自然滚动和现有虚拟化。
- 分页模式支持按钮、左右方向键和点击阅读区边缘翻页；交互控件和文本选择不触发翻页。
- 分页模式支持三种过渡；滚动模式不使用 page-curl。
- 切回滚动模式时保持相同 charOffset，不能跳到章节首部。

## 6. PDF 无缝滚动与动画

### 6.1 连续模式

继续使用当前 `PdfReaderAdapter` 的 PDF.js display layer，不整体嵌入通用 PDF Viewer：

- 用 TanStack Virtual 建立页面虚拟列表，页面原始宽高作为估算尺寸。
- 可见范围加前后各 1 页 overscan；Canvas、文本层和标注层只为可见/overscan 页面存在。
- render task 按 page 管理；页面离开渲染窗口时取消未完成任务并释放大 Canvas。
- 当前页取 viewport 中心线附近页面，进度保存 `page + pageOffsetRatio + scale/zoomMode`。
- 缩放、适合宽度和窗口调整后使用页内比例做 scroll anchoring。
- 目录、搜索、书签和标注跳转滚动到目标页及页内 rect；跨页文本选择仍不进入 v0.2。

长文档验收以 500 页生成 fixture 为基准：稳定滚动时不得同时保留整本文档 Canvas，DOM 中高成本页面层数量保持在可见页加 overscan 范围内。

### 6.2 PDF 翻页动画

- single/double 模式支持 `none`、`slide`、`page-curl`。
- continuous 模式忽略保存的 transition 并自然滚动。
- page-curl 只动画已渲染 Canvas 展示层，真实文本层和标注层在动画完成后恢复交互。
- double 模式按 spread 前进/后退，奇偶页和末页必须稳定。

## 7. UI 渐进统一

不改变书架直达、阅读器侧栏、底部导航和主题面板的核心信息架构。视觉方向继续使用现有炭黑、琥珀、青绿和纸张色，先统一再美化。

阶段 9 开始后、任何 UI 代码修改前，必须生成并审批四组完整概念：

1. 1280×800 有书书架，包含 grid/list、导入和书籍操作。
2. 1280×800 阅读器，覆盖 TXT/EPUB/PDF 共用 chrome、侧栏和设置。
3. EPUB 图片查看器及缩放状态。
4. 375×760 书架、阅读器抽屉和关键模态状态。

概念批准后提取：

- 颜色、排版、间距、圆角、边框、阴影、层级和动效时长 token。
- 按钮、分段控件、工具栏、侧栏、状态、模态框、滑杆和图标规范。
- 桌面、640px 最小窗口和 375px 测试视口的布局规则。
- 浅色、深色、纸色、护眼色下的应用 chrome 与正文关系。

`ReaderShell` 按应用壳、侧栏、选择/批注浮层、格式阅读器、设置和导航控制器拆分。每次拆分必须保持行为和测试，禁止先重写再补回归。

## 8. 后续实施顺序

### 阶段 9：阅读体验基础与设计系统

- 建立 `codex/v0.2.0-integration`。
- 增加共享类型、默认值、能力描述和 `app_settings` 持久化。
- 实现动画控制层原型并完成 page-flip 决策门槛。
- 审批 UI 概念、建立 token，并开始无行为变化的 ReaderShell 拆分。

### 阶段 10：EPUB 增强

- 出版物 page-list 与 Location 回退。
- 图片查看器。
- EPUB single/double 翻页动画。

### 阶段 11：TXT 分页

- 测量分页器、分页缓存和三页窗口。
- 滚动/分页模式切换。
- TXT 分页动画和稳定 locator 恢复。

### 阶段 12：PDF 连续模式

- 连续页面虚拟化和页内恢复。
- 搜索、目录、标注跳转整合。
- PDF single/double 翻页动画。

### 阶段 13：产品收口与数据安全

- 渐进 UI 统一和完整视觉回归。
- 版本化 JSON 备份/恢复；默认包含书库元数据、设置、进度、书签和标注，原书文件作为显式可选项。
- 评估并接入 Tauri updater；代码签名单独作为发布轨道，证书不可用时不能阻塞其他功能开发。
- 书籍元数据/封面编辑和文件夹/拖放批量导入列为 v0.2 should-have。

### 阶段 14 及以后

- MOBI/AZW3：优先评估导入时转换为 EPUB；转换器许可、体积、离线性和分发方式未通过前不进入默认安装包。
- macOS/Linux：先完成路径、权限、WebView、打包和文件关联矩阵，再承诺发行。
- 移动端：只共享纯 TypeScript 数据模型、locator、导出格式和业务规则；DOM、Tauri、epub.js、PDF.js adapter 留在平台层。
- TTS、词典/翻译、云同步和移动客户端进入 v0.3+，不扩大 v0.2 范围。

## 9. 补充功能优先级

| 优先级 | 能力 | 结论 |
|--------|------|------|
| v0.2 must | 用户要求的翻页动画、EPUB 页码/图片、TXT 分页、PDF 连续滚动 | 进入阶段 9–12 |
| v0.2 must | 完整备份与恢复 | 在更多设置和 locator 扩展后提供数据安全出口 |
| v0.2 should | 应用内更新、元数据/封面编辑、文件夹/拖放导入 | 进入阶段 13，不能阻塞核心阅读能力 |
| v0.2 could | 自定义字体、全书库全文索引、阅读历史和统计 | 核心功能完成后按剩余容量选择 |
| v0.3+ | TTS、词典/翻译、云同步、移动端、MOBI/AZW3 | 延后，避免架构和发布范围同时膨胀 |

## 10. 验收矩阵

### 动画

- EPUB、TXT 分页、PDF single/double 均覆盖 none/slide/page-curl。
- 快速重复输入最多保留一个待处理导航，进度每次只提交一次。
- reduced motion、渲染失败和调整布局时无动画安全降级。
- 选择、书签、搜索、批注和焦点不因展示层丢失。

### EPUB

- 数字/罗马数字 page-list、href/CFI 边界、有/无/损坏 page-list。
- 无出版物页码时只显示 Location，不显示估算 Page。
- Raster/SVG 图片、键盘打开、100%–500% 缩放、拖动、重置、Esc 和焦点恢复。

### TXT

- 滚动/分页切换保持 charOffset。
- 主题、窗口和 DPI 改变后回到相同文本附近。
- 大型 TXT 只挂载有限页面/段落，标注范围和跳转正确。

### PDF

- 500 页 fixture 连续滚动不创建整本文档 Canvas。
- 缩放和窗口变化保持 pageOffsetRatio 锚点。
- 目录、搜索、书签、批注跳转与连续模式协作。
- single/double/continuous 切换和末页奇偶处理正确。

### UI 与发布质量

- Browser 优先完成真实页面 QA；Playwright 负责 seeded 阅读状态。
- 覆盖 1280×800、900×640、640×640、375×760 和 DPR 2。
- 浅色、深色、纸色、护眼色均无溢出、裁切或不可读控件。
- axe 无 serious/critical；键盘和 reduced motion 路径可用。
- 每个实施阶段合入前运行 core build、lint、Vitest、Rust tests、Playwright；阶段 13 完成后再跑 Tauri release build。
